=====================================
ESSENTIAL FILES BREAKDOWN
=====================================

1. DATABASE & CONFIGURATION
---------------------------
config.php              - Database connection, session management
db_structure.sql        - Database schema with 3 tables
                         (users, events, registrations)

2. CORE PAGES
---------------------------
index.php              - Home page with features & stats
events.php             - List all events with search/filter
event_details.php      - Event details + registration form
dashboard.php          - Admin/Student dashboard

3. AUTHENTICATION
---------------------------
login.php              - User login with validation
register.php           - New user registration
logout.php             - Session destruction

4. ADMIN FUNCTIONS
---------------------------
create_event.php       - Create new events (provided)
edit_event.php         - Modify existing events
delete_event.php       - Remove events
view_registrations.php - See who registered for events

5. COMPONENTS
---------------------------
includes/navbar.php    - Responsive navigation
includes/footer.php    - Footer with links
assets/css/style.css   - Custom styling
assets/js/validation.js - Form validation


=====================================
HOW TO CREATE MISSING FILES
=====================================

For EDIT_EVENT.PHP:
- Copy create_event.php
- Add event_id parameter to fetch existing data
- Prepopulate form fields
- Change INSERT to UPDATE query

For DELETE_EVENT.PHP:
- Get event_id from URL
- Confirm deletion
- Execute DELETE query
- Redirect to events.php

For VIEW_REGISTRATIONS.PHP:
- Get event_id from URL
- SELECT registrations with JOIN
- Display in table format
- Add export option

For MY_REGISTRATIONS.PHP:
- Copy student section from dashboard.php
- Show only current user's registrations
- Add cancel registration option
